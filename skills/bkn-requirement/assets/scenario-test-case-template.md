# 业务验收用例模板

验收用例必须使用业务输入和业务期望，不要求业务用户理解对象、关系、指标、算子或 BKN 术语。

## 表格形式

| ID | 场景 | 用例类型 | 用户角色 | 用户输入 | 期望业务输出 | 必需证据 | 通过标准 | 边界 / 反例条件 |
|---|---|---|---|---|---|---|---|---|
|  |  |  |  |  |  |  |  |  |

## 用例类型说明

| 类型 | 说明 |
|---|---|
| 业务提问 | 业务用户会直接询问的问题。 |
| 分析请求 | 需要跨系统、跨规则或多步判断的分析任务。 |
| 决策建议 | 需要给出建议、依据和风险边界的任务。 |
| 草案生成 | 只生成草案或建议，不直接执行高风险动作。 |
| 权限边界 | 无权限或高风险时应拒绝、转审批或要求确认。 |
| 数据不足 | 数据不足、冲突或未对齐时应说明原因。 |
| 证据解释 | 要求输出证据、来源和推理过程。 |

如需技术测试信息，应在 `BKN_Creator` 交接之后由下游建模阶段补充，不写入业务 PRD 主体。
