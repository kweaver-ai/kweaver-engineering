# BKN_Creator 中文交接摘要模板

需求发现阶段只输出业务语言。PRD 正文末尾的 `BKN_Creator` 交接摘要，用于帮助后续建模理解业务，不承担机器可读 schema 职责。

## 按场景收敛摘要

> 五层表达必须逐项业务化。不要只写“概念模型层有哪些、关系层有哪些”，而要说明每个对象、关系、逻辑、治理点和 Agent 任务是什么、为什么需要、怎么做或谁负责。

#### S1 <场景名称>

##### 概念模型层：需要识别的业务对象

> 本体建模门禁：对象必须是稳定业务实体或事实载体；需说明身份、生命周期、责任、关系或管理动作；不得把编码、字段口径、统计维度、余额、数量、指标、结果、明细、行、报表直接当对象。
> 核心对象优先门禁：先识别核心业务对象，再判断是否拆细分对象；粒度、状态、数量、口径、扣减项默认是属性、枚举、关系或计算口径，不能直接升格为对象。
> 规则与角色不对象化门禁：业务规则、口径、算法判断、建议结果、场景角色名默认不是对象；应先归入关系约束、逻辑、Action 草案、治理要求或已有对象的场景角色。

## 全局归并摘要

### 业务已确认内容

- 仅放入已被业务材料、规则或验收用例支撑的内容。
- 确认状态门禁：凡含“待确认 / 未确认 / 待对齐 / 是否 / 范围待定 / 范围待确认 / 候选”的句子，不得进入本节。

### 仍属建模候选

- 放入 AI 工程师根据 PRD 推导出的对象、关系、逻辑或 Action 候选。

### 需下游建模阶段判定的问题

- 放入对象边界、关系表达、逻辑挂载、Action 执行模式等后续建模问题。
